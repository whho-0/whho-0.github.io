> **선택한 직무:** 보안 분야 **ML/MLOps 엔지니어**  
> **관심 분야 확장:** 디지털 포렌식 엔지니어와 클라우드 보안 엔지니어에도 관심이 있지만, 이번 글에서는 세 분야를 연결할 수 있는 출발점으로 ML/MLOps 엔지니어를 선택하였다. 머신러닝으로 보안 위협을 탐지하고, 모델을 실제 서비스에서 안정적으로 운영하는 역량은 포렌식 데이터 분석과 클라우드 보안 자동화에도 활용될 수 있다고 생각한다.

## 1. 선택한 직무 정의

보안 분야 ML/MLOps 엔지니어는 인공지능과 머신러닝을 이용하여 악성코드, 수상한 네트워크 트래픽, 시스템 로그 등에서 보안 위협을 찾아내는 모델을 만들고 운영하는 사람이다. 단순히 모델의 정확도를 높이는 데서 끝나지 않고, 실제 보안 서비스에서 모델이 계속 잘 동작하도록 배포·모니터링·개선까지 담당한다.

쉽게 말하면 **보안 데이터를 보고 위험 신호를 찾아내는 AI를 만들고, 그 AI가 현장에서 계속 믿을 수 있게 관리하는 엔지니어**라고 할 수 있다. 안랩의 ML/MLOps 직무는 AI/ML 모델의 설계·개발·최적화, 위협 탐지·분석·대응을 위한 데이터와 모델 구축, 모델 성능 모니터링 및 지속적 개선을 주요 업무로 제시한다. 

### 주요 업무

1. **보안 데이터 수집·전처리:** 네트워크 패킷, 서버·보안 로그, 파일 정보 등에서 분석에 필요한 특징을 추출하고 학습용 데이터셋을 구성한다.
2. **위협 탐지 모델 개발:** 악성 행위나 이상 징후를 분류·탐지하는 머신러닝/딥러닝 모델을 설계하고 성능을 검증한다.
3. **모델 운영(MLOps):** 학습한 모델을 서비스 환경에 배포하고, 성능 저하나 데이터 변화가 발생하는지 모니터링하여 재학습·개선한다.
4. **보안 팀과의 협업:** 탐지 결과가 실제 보안 대응에 도움이 되도록 보안 분석가와 협업하고, 개발 과정과 결과를 문서화한다.

## 2. 실제 채용공고 살펴보기

### 1) 안랩 — ML/MLOps 엔지니어

- **주요 업무:** AI/ML 모델 설계·개발·최적화, 위협 탐지와 분석·대응을 위한 데이터 구성 및 모델 구축, 모델 성능 모니터링과 지속적 개선이다. 

- **확인한 역량:** 머신러닝 모델 개발 능력뿐 아니라 보안 위협 탐지라는 도메인 문제를 데이터와 모델로 해결하는 역량이 중요하다.

### 2) 네이버클라우드 — MLOps 엔지니어

- **주요 업무:** 대규모 머신러닝·딥러닝 서비스의 개발, 운영, 최적화를 담당하며 여러 도메인과 서비스에서 발생하는 데이터와 모델을 다룬다. 

- **확인한 역량:** 모델을 만드는 능력 외에 대규모 서비스 환경에서 데이터와 모델을 안정적으로 운영하고 최적화하는 역량이 필요하다.

### 3) Socra AI — MLOps Engineer

- **주요 업무:** 머신러닝 모델에 대한 이해를 바탕으로 MLOps 엔지니어링을 수행하고, 모델을 서비스에 효과적으로 적용하는 역할이다. 

- **확인한 역량:** 모델 학습 지식과 소프트웨어·인프라 관점의 운영 역량을 함께 갖추어야 한다.

## 3. 공통적으로 요구되는 핵심 역량 3가지

### 1) 머신러닝·딥러닝 모델 개발과 Python 활용 능력

세 공고 모두 모델을 개발하거나 서비스에 적용하는 역할을 포함한다. 따라서 Python을 활용해 데이터를 다루고, `scikit-learn`, `PyTorch`, `TensorFlow`, Hugging Face 같은 라이브러리로 모델을 학습·평가하는 능력이 기본이 된다. 특히 보안 ML/MLOps에서는 단순히 최신 모델을 사용하는 것보다 문제에 맞는 특징, 모델, 평가 지표를 선택하는 판단력이 중요하다. 안랩 공고 역시 AI/ML 모델의 설계·개발·최적화를 핵심 업무로 제시한다.

### 2) 데이터 파이프라인과 MLOps 운영 역량

MLOps 엔지니어는 실험용 노트북에서 모델을 만드는 데서 끝나지 않고, 데이터 수집부터 학습·배포·모니터링·재학습까지의 전체 생명주기를 관리해야 한다. 대규모 ML/DL 서비스의 개발·운영·최적화를 담당하는 네이버클라우드 공고와, 모델을 서비스에 효과적으로 적용하는 Socra AI 공고는 이러한 운영 역량의 중요성을 보여준다. 

학습이 필요한 기술은 Docker와 Linux, Git/GitHub, CI/CD, 모델·실험 관리 도구, API 서버 구축, 클라우드 환경(AWS·GCP 등)이다. 이후에는 Spark 같은 분산 처리와 Kubernetes 기반 배포까지 확장해 보고 싶다.

### 3) 보안 도메인 지식과 탐지 결과 해석 능력

보안 분야의 모델은 일반적인 이미지·추천 모델과 달리 공격자의 행위와 시스템 환경을 이해해야 제대로 설계할 수 있다. 안랩은 실제 보안 위협을 탐지하고 대응하는 서비스를 위한 AI/ML 역할을 제시하며, 위협 탐지·분석·대응용 데이터와 모델 구축을 업무로 둔다. 

따라서 네트워크 기본 구조와 프로토콜, Linux·Windows 로그, IDS/IPS, 악성코드와 침해사고 대응의 기초를 학습해야 한다. 또한 모델이 경고를 발생시킨 이유를 보안 담당자가 이해할 수 있도록 설명하고, 오탐·미탐의 영향을 분석하는 능력도 중요하다.

## 4. 이번 여름방학 목표

이번 방학에는 “**보안 데이터를 활용한 ML 모델을 만들고, 작은 서비스 형태로 운영해 보기**”를 목표로 삼고 싶다. 공부만 나열하기보다 결과물을 GitHub와 블로그에 남겨 나의 역량을 증명하는 방향으로 진행할 것이다.

## 5. 마무리

ML/MLOps 엔지니어는 AI 모델 개발 능력과 서비스 운영 능력, 그리고 보안 도메인 지식을 함께 요구하는 융합 직무이다. 나는 이번 여름방학 동안 보안 로그 또는 네트워크 데이터를 활용한 탐지 프로젝트를 완성하고, 이를 Docker와 API 형태로 배포해 보는 것을 가장 우선적인 목표로 삼겠다. 이 경험을 바탕으로 이후에는 디지털 포렌식의 증거·로그 분석, 클라우드 환경의 보안 모니터링으로 관심 분야를 넓혀 가고 싶다.
